Architecture (3-Tier)

🖥️

Frontend (React + Vite)

React 19 + Vite + TypeScript + Tailwind CSS v4 + ShadCN/UI + Framer Motion. Role-based dashboards, code-split pages, Socket.io client, Recharts analytics. Dark/light theme support.

⚡

MongoDB Atlas with 11 Mongoose models. GeoJSON indexes for spatial queries. Immutable audit logs. Flask ML microservice for hotspot prediction.

What the System Does

📋

Crime Reporting & Case Management

Citizens report crimes online. CSOs create FIRs (First Information Reports) with auto-generated case numbers (FIR-YYYY-NNNN). Cases flow through an 8-stage lifecycle from pending to closed, with status transition validation enforced at the database level.

🔍

Criminal Records & ID Verification

Officers verify citizens using Fayda (Ethiopia's national ID). The system cross-references against the criminal records database to flag wanted/missing persons. Full-text search on names, with criminal history tracking per record.

📸

Evidence Chain of Custody

Digital evidence management with QR code tracking. Every transfer between officers is logged in an append-only chain. Supports file upload (photos, video, documents) to local storage or Cloudinary CDN. 8 evidence lifecycle statuses.

🚨

Alert Broadcasting

Commanders broadcast alerts to all officers, specific stations, or individual recipients. Officers acknowledge alerts in real-time via WebSocket. Supports crime, missing, wanted, and emergency alert types with priority levels.

🗺️

AI-Powered Hotspot Prediction

Flask ML microservice analyzes crime location data using K-Means clustering, DBSCAN density analysis, and Random Forest prediction. Generates risk zones on the Adama city map. Auto-falls back to cached data if ML service is offline.

🚔

Patrol Tracking

Officers start patrols with GPS tracking. Location updates stream via WebSocket to command dashboards. Route history stored as lat/lng/timestamp arrays. Duration computed automatically. Ownership checks prevent IDOR attacks.

📈

Analytics & Reporting

Dashboard KPIs (case counts, active officers, patrol status) available to all roles. Commanders and admins generate daily/weekly/monthly reports with aggregated crime stats, patrol metrics, and alert summaries. Reports exportable as CSV or JSON.

📜

Immutable Audit Trail

Every significant action is logged to an immutable audit_logs collection. Database hooks block all update/delete operations. Tracks user, action, entity, IP address, and timestamp. 20+ audited action types across all modules.

What Happens During Investigation (Step 4)

📸 Evidence Collection

Officers upload photos/documents via Multer. Each piece gets a unique QR code. Files go to Cloudinary CDN (or local storage).

🔗 Chain of Custody

Every evidence transfer is logged: who, when, action, notes. Append-only — entries can never be edited or deleted.

🔍 Suspect Linking

Investigators link criminal records to cases via national ID. The system cross-references Fayda verification with criminal database.

👤 Person Records

Victims, witnesses, and complainants are recorded with statements and linked to one or more cases.

📝 Investigation Notes

Officers add timestamped notes (general, investigation update, evidence note, status change) visible on the case timeline.

📜 Audit Trail

Every view, create, update, and status change is logged immutably with user ID, IP, and timestamp.

Middleware Chain

Every request flows through this pipeline. Order matters — each layer validates before passing to the next.

1
Rate Limiter

100 req/15min per IP (global). Stricter: auth 5/15min, reports 3/hr, tracking 10/hr

2
CORS

Whitelist allowed origins. Credentials enabled for cookie/token passing.

3
Helmet

Sets security headers: CSP, X-Frame-Options, HSTS, etc.

4
Body Parser

JSON (10MB limit) + URL-encoded. Multer for multipart file uploads.

5
JWT Authentication

Verifies Bearer token. Checks blacklist. Loads user + station. Attaches to req.user.

6
RBAC Authorization

Checks req.user.role against allowed roles for the endpoint. Returns 403 if denied.

7
Validation

express-validator checks request body/params. Returns 400 with field-level errors.

8
Controller

Business logic. Database operations. Emits real-time events. Returns response.

9
Audit Logger

Logs action to immutable audit_logs collection. Captures user, IP, entity, timestamp.

Security Features

  • Passwords hashed with bcrypt (12 rounds)
  • JWT access tokens (24h) + refresh tokens (7d) with rotation
  • Token blacklist on logout (auto-cleanup)
  • Rate limiting per IP (global + per-route)
  • Helmet security headers
  • CORS origin whitelist
  • Password reset via crypto tokens (30min expiry)
  • Patrol ownership check (prevents IDOR)
  • Immutable audit logs (all mutations blocked)
  • Input validation on all write endpoints

Real-Time Events (Socket.io)

WebSocket server runs alongside the HTTP server. Authenticated users join rooms based on their role, user ID, and station. Events are emitted from controllers after database writes.

Connection & Rooms

🔐 Auth

Clients pass JWT via socket.handshake.auth.token. Invalid/missing tokens connect as public (limited events).

🏠 Room Assignment

Each user auto-joins: user:{id}, role:{role}, station:{stationId}, and public.

📡 Targeting

Events target specific rooms — e.g. alerts go to each recipient's user room, patrol updates go to station + commander rooms.

Events

alert:new New alert broadcast — fired when commander/admin/CSO creates an alert → Each recipient's user:{id} room
alert:critical Critical priority alert — broadcast to everyone → All connected clients
case:updated Case status changed or assigned — includes case number, new status, update type → Assigned officers + station + admin/commander
patrol:location GPS update from active patrol — lat, lng, timestamp → Station room + commander/admin
patrol:event Patrol started or ended → Station room + commander/admin
public:new_report Citizen submitted a new crime report — notifies CSOs for review → role:cso + role:admin

Flask ML Microservice (Python)

Separate Python service on port 5001. Reads case data from MongoDB, trains ML models, and writes predictions back to the hotspots collection.

AlgorithmPurposeOutput
K-MeansCluster crime locations into hotspot zonesCluster centers + case counts + risk scores
DBSCANDensity-based clustering (finds organic patterns)Dense clusters + noise points

Node.js controller calls Flask via HTTP. If ML service is offline, returns cached hotspot data from MongoDB.
